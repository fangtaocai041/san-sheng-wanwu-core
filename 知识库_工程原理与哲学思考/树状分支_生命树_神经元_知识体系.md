# 树状分支：生命之树 × 神经元树突 × 知识体系

> 生命之树、神经元树突、知识分类树——同一个深层模式的三次独立发现。
> 达尔文用一棵树概括了所有物种的亲缘关系；
> 神经元用一棵树接收并处理来自成千上万个上游的信号；
> 我们用一个树状架构组织知识：从域到界到门到纲到目到科到属到种。

---

## 1. 三棵树的同构性

| 特征 | 生命之树（进化） | 神经元树突（计算） | RCCA（知识架构） |
|:-----|:---------------|:-----------------|:----------------|
| **根** | 共同祖先 LUCA | 胞体 (Soma) | Pipeline 总入口 |
| **主干** | 主要演化支 | 顶树突 (Apical) | 核心认知流程 |
| **分支** | 目/科/属分化 | 基底树突 (Basal) | 12 学科 MoE 路由 |
| **叶节点** | 现存物种 | 突触后末梢 | 具体知识条目 |
| **分支特异性** | 不同生态位适应 | 不同分支处理不同输入模式 | 不同专家处理不同领域 |
| **修剪** | 灭绝 | 突触修剪 (Pruning) | 遗忘曲线淘汰 |
| **生长** | 辐射适应 | 分支延伸+新突触形成 | TranspositionLayer 转座 |
| **局部可塑性** | 种群局部适应 | 分支特异性可塑性 | domain-specific 学习率 |

## 2. 关键发现（2025-2026）

### 2.1 树突不是被动电缆——是分布式计算网络

**Poirazi 实验室 (Nature Communications 2025)**:
> "树突赋予人工神经网络准确、鲁棒且参数高效的学习能力。"
> —— Chavlis & Poirazi, 2025

传统观点：树突只是把信号"传"到胞体。
现代观点：**每个树突分支是一个独立的计算单元**，可以执行 AND/OR/XOR 等逻辑运算，
有自己的局部可塑性规则，独立于其他分支的权重更新。

### 2.2 树突分支的计算复杂度（NeurIPS 2025）

> "树突树的大小直接决定了神经元能计算什么函数。"
> —— Agrawal & Buice, NeurIPS 2025

- **基底样架构** (Basal-like): 功能实现的概率更高（更鲁棒）
- **顶树突样架构** (Apical-like): 快速重训练的优势更强（更灵活）
- 两者是**不相交的函数集**——不同分支类型做不同的事

### 2.3 分支特异性可塑性 (Nature Neuroscience 2025)

> "间隔化树突可塑性将时间上相近的情景记忆链接在一起。"
> —— Sehgal et al., 2025

同一神经元的不同分支可以独立地编码不同的记忆，并且可以
在分支层面实现记忆的融合。

### 2.4 进化树的分支法则

达尔文的树不仅是分类工具——它揭示了**层次化模块化**的核心工程原理：
- 每个分支点的决策是局部的（只是分化成两个新物种）
- 但全局结构是递归的（分支上再生分支）
- 适应性 = 分支模式与环境结构的匹配度

## 3. 工程原理清单

### 原理 1：分支特异性计算

**生物学**: 每个树突分支可以独立执行 AND/OR/XOR 等逻辑运算。
**工程映射**: MoE 专家的每个专家应该有独立的路由决策和局部可塑性。

**RCCA 改进**:
- 当前 KnowledgeRouter 对所有专家使用相同的匹配逻辑
- 改进: 每个专家维护自己的 `local_learning_rate`，根据历史成功率动态调整
- 参考: Poirazi Lab 2025 "dendritic compartment-specific plasticity"

### 原理 2：基底 vs 顶树突分工

**生物学**: 基底树突更鲁棒（高实现概率），顶树突更灵活（快速重训练）。
**工程映射**: 系统需要两种互补的认知模式。

**RCCA 改进**:
- **基底模式** (Basal): Pipeline Phase 1-3 的标准感知-认知流程（鲁棒、确定）
- **顶树突模式** (Apical): ReflectionLoop + TranspositionLayer（灵活、可塑）
- 两种模式共享 SelfModelEngine 作为"胞体"，但使用不同的处理路径
- 参考: NeurIPS 2025 Agrawal & Buice

### 原理 3：局部→全局整合的层级

**生物学**: 树突分支局部计算 → 分支汇总到胞体 → 轴突全局输出。
**工程映射**: 知识处理应该分层：局部特征提取 → 领域内合成 → 跨域综合。

**RCCA 改进**:
- 当前 pipeline 只有平行感知 + 线性认知
- 改进: 加入明确的**分支汇聚步骤**:
  ```
  输入 → 分支感知 (5 senses 并行)
       → 分支认知 (每个分支独立 dialectics)
       → 分支汇聚 (胞体级 RRF 融合)
       → 全局输出
  ```
- 参考: Poirazi & Mel (2001) "2-layer neural network" 模型

### 原理 4：生长 + 修剪的平衡

**生物学**: 树突不断生长新分支，同时修剪未使用的分支。
**工程映射**: 知识图谱应该持续扩展，同时淘汰无效节点。

**RCCA 改进**:
- TranspositionLayer 的转座 = "生长"新连接
- `consolidate.py` 的遗忘曲线 = "修剪"无用连接
- 需要添加一个**修剪阈值**: 如果某个转座通路在 N 次尝试后 fitness < 阈值 → 自动裁剪
- 参考: Cuntz et al. (2010) "One rule to grow them all"

### 原理 5：亲缘关系决定共享属性

**生物学**: 同一分支内的物种共享更多特征（同源结构）。
**工程映射**: 同一学科拓扑分支内的概念应该共享推理模式。

**RCCA 改进**:
- 当前 12×12 拓扑矩阵是静态的
- 改进: 如果"生物学→生态学"的转座被驯化了，则"生物学→保护生物学"的转座偏置也应该增加
  （相当于进化树上相邻分支间的转移概率更高）
- 参考: 进化树的"系统发育信号" (phylogenetic signal)

### 原理 6：分支点的不可逆性

**生物学**: 一旦两个物种从分支点分开，通常不会重新融合。
**工程映射**: 一旦两个学科的知识领域分化，它们不应被强制统一。

**RCCA 改进**:
- MoE 路由的专家应该保持**独立的知识空间**（不共享嵌入）
- 跨域工作应该通过转座（复制+移动）而非融合（合并+模糊界限）
- 参考: Dollo 不可逆法则 (演化不会走回头路)

## 4. 架构优化建议

### 4.1 当前 Pipeline 的问题

```
当前: senses → cortex → motor (线性, 无分支汇聚)
```

### 4.2 建议的树突式 Pipeline

```
                    ┌─ Scholar ──→ 分支认知 ──┐
                    ├─ CNKI   ──→ 分支认知 ──┤
  输入 ─→ 胞体决策 ──┼─ NCBI   ──→ 分支认知 ──┼─→ 分支汇聚 ─→ 全局输出
                    ├─ FishBase → 分支认知 ──┤
                    └─ Web    ──→ 分支认知 ──┘
                          ↑             ↑
                    每个分支独立      RRF 融合
                    可塑性规则        (胞体级整合)
```

### 4.3 需修改的文件

| 文件 | 改动 |
|:-----|:------|
| `cortex/pipeline.py` | Phase 2 拆分为"分支认知" + "分支汇聚"两步 |
| `cortex/dialectics.py` | 添加分支级别的可塑性（不同 domain 不同 learning rate）|
| `memory/kb/search.py` | KnowledgeRouter 增加"亲缘传播"（驯化通路提升邻近域权重）|
| `memory/consolidate.py` | 添加修剪阈值：低 fitness 的转座通路自动裁剪 |
| 知识库文档 | 本文件已记录全部 6 条原理 |

## 5. 与球体哲学的融合

**球体"三心合一" + 树突"分支汇聚" = 完整的认知架构**:

```
球体（宏观）:  三心合一 → 逻辑中心 = 数据重心 = 价值核心
                  ↓
树突（微观）:  分支汇聚 → 多源输入 → 胞体整合 → 轴突输出
                  ↓
RCCA（实现）:  5 senses 并行感知 → Pipeline 汇聚 → 全局决策
```

- **球体**告诉你：系统应该是**各向同性**的（无偏见处理）
- **树突**告诉你：系统应该是**分支特异性**的（每个源按其特性独立处理）
- 两者不矛盾——输入阶段要"分"（树突），汇聚阶段要"合"（球体三心）
