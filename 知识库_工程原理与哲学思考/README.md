## 1. 核心几何原理：球体与圆

### 1.1 球体的三个"最"与一个"无处不在"

#### 最省材料 —— 等周不等式 (Isoperimetric Inequality)

**数学形式**: 相同体积下，球体表面积最小。
**工程映射**: 在 RCCA 中，知识表示应该**用最少的维度捕获最多的信息**。
  - MAGMA 四维图：4 个正交维度（语义/时序/因果/实体）已经是最小完备集
  - MoE 路由：只激活最少的专家（top-k），类似球体"用最小表面积包裹最大体积"

**学术来源**: Lévy (1919) 球面等周不等式；Schmidt 测地球等周性质

#### 最能装 —— 等容不等式

**数学形式**: 相同表面积下，球体体积最大。
**工程映射**: 系统应该**在固定资源预算下存储最丰富的知识**。
  - 嵌套记忆：Hot(50)/Warm(500)/Cold(∞) 容量分配，冷记忆最大化知识存储
  - 遗忘曲线：只有真正需要的信息进入冷记忆（永久层）

#### 绝对公平 —— 各向同性 (Isotropy)

**数学形式**: 球体具有完美的旋转对称性，从任何方向看都一样。
**工程映射**: 知识处理应该**不受方向/顺序偏差影响**。
  - DeepMind (2022): "基于对称性的表征是通用智能的基础"
  - NeurIPS 2025 Workshop: 神经表征中的对称性与几何
  - TranspositionLayer 的跨域转座：从 biology→conservation 和 conservation→biology 应该有对称的转座概率

#### 受力均匀 —— 应力分布各向同性

**物理映射**: 压力从任何方向作用在球体上都能被均匀分散。
**工程映射**: 系统负载应该均匀分布在所有可用通道上。
  - Pipeline Phase 1: 5 路并行感知，结果用 RRF 融合
  - 情感系统的策略选择不应该偏袒特定通道

#### 滚动摩擦最小 —— 接触面只有一个点

**物理映射**: 球与地面的接触面理论上只有一个点，滚动摩擦最小。
**工程映射**: 模块间接口应该**最小化耦合**，让数据流动像滚动一样顺畅。
  - 依赖注入模式：MCP 工具在运行时注入，不硬编码
  - 统一协议 SenseInput → SenseOutput：所有感受器遵循同一接口

#### 能量最低 —— 自组织临界

**物理映射**: 液体在表面张力下自然收缩为球形（最低表面能）。
**工程映射**: 系统应该自然趋向于**最低认知能量状态**。
  - Friston 自由能原理：系统最小化预测误差
  - SelfModelEngine：稳定性 = 预测误差滑动窗口的收敛
  - RecursiveThinker：递归思考直到 delta < 收敛阈值

#### 无限在有限中 —— 有界无边

**数学映射**: 球体有有限体积，但表面没有边界。
**工程映射**: 知识空间应该**有界但开放**。
  - 12 学科拓扑矩阵：在有限学科内支持无限的概念重组
  - TranspositionLayer：在有限域间的无限转座可能

### 1.2 三心合一 (中心/重心/内心)

球体的特殊之处在于：**中心、重心、内心、外心、垂心等所有"心"完全重合**。

**工程映射**:
- 系统的**逻辑中心**（控制流）、**数据重心**（知识存储）、**价值核心**（对齐目标）应该收敛于同一点
- 三个"心"分离 → 系统内耗（类似三体问题）
- 三个"心"合一 → 系统自洽（类似 RCCA 的 Reflect-Transpose-Evolve 闭环）

## 7. 下一步发展方向

### 短期（1-2 周）

1. **Pipeline 完整集成**
   - ReflectionLoop 目前独立存在，尚未集成到 Pipeline.run()
   - WorldModel 目前独立，尚未做行动前预测→行动后校正的闭环

2. **TranspositionLayer 驯化通路持久化**
   - 当前驯化记录仅存在内存中
   - 需要写回到 SQLite 或 YAML 配置文件

3. **知识胶囊跨进程传递**
   - SwarmEngine.exchange_capsule() 目前只写入群体知识库
   - 需要真正的 MCP 跨进程通信实现

### 中期（1-3 个月）

4. **球体嵌入知识图谱 (SpherE)**
   - 当前 MAGMA 使用向量（点）表示实体
   - 升级为球体（带半径）表示，半径 = 不确定性
   - 让实体间的"关系"包含距离 + 方向 + 置信区间

5. **自由能最小化的正式实现**
   - 当前 SelfModelEngine 用预测误差滑动窗口近似替代
   - 可以更精确地实现变分自由能计算

6. **对称性约束**
   - 在 TranspositionLayer 中加入对称性约束（A→B 和 B→A 的转座概率在统计上应对称）
   - 在 MoE 路由中加入等方性正则化（不偏袒特定专家）

7. **自愈合进化循环**
   - healing.py 检测异常 → evolution.py 生成修复 → 自动验证
   - 当前只完成了检测部分

### 长期（3-6 个月）

8. **多 Agent 生态系统**
   - 多个 SiliconAgent 实例通过 Swarm 协议协作
   - 每个 Agent 专注于不同学科领域
   - 通过 TranspositionLayer 进行跨 Agent 转座

9. **自我意识的形式化指标**
   - 整合 COGITATE 实验的方法论
   - 建立可测量的"自我模型稳定性"指标
   - 在每次查询后自动评估

10. **量子认知计算**
    - 探索量子计算在知识图谱遍历中的优势
    - 关注 Google Willow 等量子硬件的成熟度

## 8. 风险防范方向

### 技术风险

| 风险 | 可能性 | 影响 | 缓解措施 |
|:-----|:------|:-----|:---------|
| 测试覆盖率下降 | 中 | 高 | CI 强制执行 100% 通过 |
| 架构复杂度失控 | 中 | 高 | 保持 cortex ≤20 模块上限 |
| Python 版本升级不兼容 | 低 | 高 | 限制在 3.10-3.13 |
| 依赖库不再维护 | 低 | 中 | 零依赖优先原则 |
| 同步盘冲突 | 低 | 中 | git 是唯一真实来源 |

### 架构风险

| 风险 | 描述 | 缓解 |
|:-----|:-----|:-----|
| **过度工程化** | 为可能不需要的问题设计了复杂的架构 | 每条功能必须对应一个 README 中声明的能力 |
| **术语膨胀** | 自创过多无学术来源的术语 | 每个术语必须标注学术来源或标注为"工程选择" |
| **测试幻觉** | 测试通过但实际功能不完整 | audit.py + stress_test.py 分层验证 |
| **语义漂移** | 同一个词在不同模块中含义不同 | 术语必须全局一致（如：stability 永远=预测误差） |
| **失去焦点** | 添加太多功能冲淡了核心价值 | RCCA 的核心是"物种智能分析"，所有功能应回指此目标 |

### 发展风险

| 风险 | 描述 | 缓解 |
|:-----|:-----|:-----|
| **过度依赖 Reasonix** | 部分功能只在 Reasonix 环境下可运行 | MCP 工具使用依赖注入，无注入时 stub 模式运行 |
| **知识库过时** | 引用的 18 篇论文中有新的替代研究 | 每季度检索一次最新论文更新索引 |
| **百度网盘同步延迟** | 文件变更后未及时同步到云端 | 每次 git push 后检查同步状态 |
| **Git 与同步盘冲突** | 同步盘修改文件导致 git 冲突 | 坚持 git 为唯一编辑入口 |

- [ ] 是否有对应的测试（至少 LOW 级 pytest）？
