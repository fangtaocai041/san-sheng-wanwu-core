## 1. 语言学：知识如何编码

### 1.1 核心洞见

语言不是知识的"容器"——它是知识的结构本身。改变语言的编码方式，就改变了可被表达的知识的边界。

### 1.2 关键理论

| 理论 | 提出者 | 核心论点 | 工程映射 |
|:-----|:-------|:---------|:---------|
| **言语行为理论** | Austin (1962) | 说话即行动：陈述/疑问/命令/承诺都是不同行为 | SenseInput 需要区分查询类型（不只是 query string）|
| **语义场理论** | Trier (1934) | 词汇的意义由它在语义场中的位置决定，而非孤立定义 | MAGMA 四维图的语义关系应该包含"反义/同义/上下位" |
| **深层结构 vs 表层结构** | Chomsky (1965) | 同一深层结构可以有多重表层表达 | 不同来源的同一事实应归一化到同一深层结构 |
| **萨丕尔-沃尔夫假说** | Whorf (1956) | 语言结构影响认知方式 | 中文/英文文献的差异不仅是语言差异，可能是认知差异 |

### 1.3 对 RCCA 的改进

**当前**: SenseInput 只有 `query: str`，不分言语行为类型。
**改进**: 区分查询的语用类型：

```python
class SpeechAct:
    ASSERTION = "assertion"      # "鳤是洄游鱼类"（陈述事实）
    QUESTION = "question"        # "鳤为什么濒危？"（寻求解释）
    DIRECTIVE = "directive"      # "找到关于鳤的所有论文"（指令）
    COMMISSIVE = "commissive"    # "我将验证这个结论"（承诺）
    EXPRESSIVE = "expressive"    # "这个结果很可疑"（表达态度）
```

不同类型触发不同的 pipeline 路径：
- QUESTION → 标准感知-认知-行动
- ASSERTION → 优先验证（检查一致性）
- DIRECTIVE → 广度优先搜索
- EXPRESSIVE → 进入猜疑链检测

## 2. 逻辑学：推理是否有效

### 2.1 核心洞见

当前 dialectics.py 只做"这些声明是否矛盾"，不做"这个推理是否有效"。
三段论、假言推理、模态逻辑——这些是推理有效性的基础工具，目前完全缺失。

### 2.2 关键理论

| 分支 | 核心内容 | 工程映射 |
|:-----|:--------|:---------|
| **三段论** | 大前提→小前提→结论 | 从物种 KB 推导隐含知识 |
| **命题逻辑** | AND/OR/NOT/IMPLIES | Pipeline 决策逻辑的形式化 |
| **谓词逻辑** | ∀/∃ 量词 | 知识库查询的精确化 |
| **模态逻辑** | □必然/◇可能 | 可信度评估的概率语义 |
| **模糊逻辑** | 部分真值 [0,1] | 可信度本身就是模糊真值 |
| **非单调逻辑** | 新信息可撤销旧结论 | 矛盾检测后的信念修正 |

### 2.3 核心示例：三段论形式化

```
当前: "鳤是鱼类" 和 "鱼类需要水" 同时在 KB 中，但没有关联

改进:
  大前提: ∀x 鱼类(x) → 需要水(x)
  小前提: 鱼类(鳤)
  结论:  需要水(鳤)

  这要求 pipeline 中增加推理步骤: 当查询"鳤需要什么环境"时，
  不是去搜索，而是先检查 KB 中是否有隐含的三段论结论。
```

### 2.4 对 RCCA 的改进

**当前**: dialectics.py 用加权投票解决矛盾，不检查推理链的有效性。
**改进**:

```python

    """三段论推理引擎"""
    def deduce(self, premises: List[str], query: str) -> InferenceResult:
        # 1. 将前提形式化为逻辑表达式
        # 2. 检查是否可通过三段论推出 query

```

这可以直接减少不必要的搜索——很多"知识"其实可以从已有知识中演绎出来。

## 3. 符号学：符号如何指向意义

### 3.1 核心洞见

MAGMA 四维图当前把所有"节点"当成同一类东西。但 Peirce 告诉我们：
符号有三种不同类型，每种指向意义的方式完全不同。

### 3.2 Peirce 三元符号模型

| 符号类型 | 定义 | 例子 | 工程映射 |
|:---------|:-----|:-----|:---------|
| **像似符 (Icon)** | 与对象相似的符号 | 图片、图表、地图 | OCR 视觉输出 |
| **指示符 (Index)** | 与对象有因果/邻近关系的符号 | 烟→火、测针→温度 | 实体之间的因果关系 |
| **规约符 (Symbol)** | 约定俗成的符号 | 文字"鱼"、€ 符号 | 知识库中的文字标签 |

**关键洞见**: 同一个实体在三类符号中的表现完全不同。
- "鳤"这个字（规约符）
- 鳤的照片（像似符）
- 鳤的 DNA 序列（指示符）

### 3.3 对 RCCA 的改进

**当前**: MAGMA 所有节点统一为 MemoryNode，没有符号类型区分。
**改进**:

```python
@dataclass
class SignNode(MemoryNode):
    """符号节点 — 知识图谱中的符号"""
    sign_type: str = "symbol"  # icon | index | symbol
    signifier: str = ""        # 能指 (符号本身)
    signified: str = ""        # 所指 (含义)
    referent: str = ""         # 指称对象 (外部世界实体)

    # 示例:
    #  sign_type = "symbol"
    #  signifier = "鳤"
    #  signified = "一种长江珍稀鱼类"
    #  referent = "Ochetobius elongatus"
```

这样，不同来源对同一实体的描述可以被正确地对齐——不管是中文名、
学名还是标本图像，它们都指向同一个 `referent`，但通过不同的 `signifier`。

## 4. 音律学：知识传递的节奏

### 4.1 核心洞见

知识传递和音乐有共同的结构：**节奏、韵律、重复、变奏。**
一个信息密度恒定的输出是难以消化的——好的知识传递应该有"呼吸"。

### 4.2 关键概念

| 概念 | 定义 | 工程映射 |
|:-----|:------|:---------|
| **节奏** | 信息的周期性变化 | Pipeline 各阶段的耗时控制 |
| **韵律** | 重音和轻音的交替 | 结论和证据的篇幅交替 |
| **重复** | 关键信息的再现 | 跨源一致信息的加权 |
| **变奏** | 同一主题的不同处理 | 同一事实的不同表达方式 |
| **休止** | 信息的中断 | 不确定时的"不知道"回应 |

### 4.3 Shannon 信息论 + 音律学

Shannon (1948) 证明信息的最优编码取决于信道的容量和噪声。
音律学告诉我们：**最优的信息节奏不是恒定的，而是与接收者的状态相关的。**

- 当系统 uncertainty 高 → 放慢输出节奏，提供更多确认
- 当系统 confidence 高 → 加快节奏，直接给出结论
- 当有多源矛盾 → 引入"休止"（不输出，触发确认性搜索）

### 4.4 对 RCCA 的改进

**当前**: pipeline 对所有查询执行相同步骤序列。
**改进**:

```python
class RhythmicOutput:
    """受音律学启发的输出节奏控制"""
    def adjust_pace(self, uncertainty: float, confusion: float) -> str:
        if uncertainty > 0.7:
            return "adagio"    # 慢板: 放慢、多确认
        elif confusion > 0.6:
            return "rest"      # 休止: 先内部解决矛盾再输出
        elif uncertainty < 0.3:
            return "allegro"   # 快板: 直接输出结论
        else:
            return "andante"   # 行板: 正常节奏
```
